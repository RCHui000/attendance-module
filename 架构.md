本文记录当前 V0.12.8.1 的系统结构、部署结构、核心数据模型和周表审批流。云服务器与 NAS 运行的是同一套应用形态，差异主要是访问入口、compose 文件和宿主机目录。

## 4. 周表审批架构

### 审批提交与流转

```mermaid
flowchart TD
  DRAFT["员工编辑周表<br/>draft / rejected"] --> SUBMIT["提交周表<br/>psa_timesheet_action(submit)"]
  SUBMIT --> LOCK["周表状态 submitted<br/>生成项目审批任务"]

  LOCK --> SPLIT["按 timesheet_entries.project_id 去重"]
  SPLIT --> P1["项目块审批任务<br/>scope_type = project<br/>scope_id = 项目ID"]
  P1 --> OWNER["项目负责人 / 项目所属部门负责人 / 部门负责人 / admin 兜底"]

  OWNER --> DECISION{"项目块审批"}
  DECISION -->|"通过"| PROJECT_DONE["该项目块 completed / approve"]
  DECISION -->|"拒绝"| REJECT["周表 rejected<br/>取消剩余 pending 任务"]

  PROJECT_DONE --> ALL_PROJECTS{"所有项目块均通过?"}
  ALL_PROJECTS -->|"否"| WAIT["等待其他项目块"]
  WAIT --> ALL_PROJECTS
  ALL_PROJECTS -->|"是"| SUMMARY["生成部门汇总审批任务<br/>scope_type = department_summary"]

  SUMMARY --> DEPT["部门负责人审批完整周表"]
  DEPT --> FINAL{"部门负责人审批"}
  FINAL -->|"通过"| APPROVED["周表 approved<br/>进入 BI 统计口径"]
  FINAL -->|"拒绝"| REJECT
```

### 审批参与者解析

```mermaid
flowchart LR
  SHEET["timesheet"] --> USER["提交员工"]
  USER --> PROFILE["employee_profiles_v2"]
  PROFILE --> DIRECT["直属负责人<br/>manager_user_id"]
  PROFILE --> ORG["员工所属部门<br/>organizations"]
  ORG --> ORG_HEAD["部门负责人<br/>organizations.manager_user_id"]

  SHEET --> ENTRIES["timesheet_entries"]
  ENTRIES --> PROJECT["projects"]
  PROJECT --> PROJECT_OWNER["project_owner_id"]
  PROJECT --> OWNER_ORG["owner_org_id"]
  OWNER_ORG --> OWNER_ORG_HEAD["项目所属部门负责人"]

  DIRECT --> DEPT_REVIEWER["部门汇总审批人"]
  ORG_HEAD --> DEPT_REVIEWER
  PROJECT_OWNER --> PROJECT_REVIEWER["项目块审批人"]
  OWNER_ORG_HEAD --> PROJECT_REVIEWER
  DEPT_REVIEWER --> PROJECT_REVIEWER
```

审批人优先级：

- 项目块审批人：项目负责人 `project_owner_id` 优先，其次项目所属部门负责人，再其次提交人部门负责人，最后 admin 兜底。
- 部门汇总审批人：员工直属负责人优先，其次员工所属部门负责人，最后 admin 兜底。
- 如果项目审批人与部门汇总审批人是同一人，系统会做折叠处理，避免同一个人重复审批同一个阶段。

## 5. BI 数据口径

```mermaid
flowchart LR
  ENTRIES["timesheet_entries<br/>项目、日期、小时"] --> FILTER["仅统计 approved / locked / summarized 周表"]
  FILTER --> EMP["员工薪酬档案<br/>employee_salary_profiles"]
  FILTER --> PROJECT["项目维度"]
  FILTER --> ORG["部门维度"]
  FILTER --> PERSON["人员维度"]

  EMP --> COST["折算人工成本"]
  PROJECT --> DASH["数据看板"]
  ORG --> DASH
  PERSON --> DASH
  COST --> DASH
```

当前 BI 关注：

- 项目维度：项目活跃数、投入工时、人工成本、项目明细。
- 部门维度：部门参与项目数、投入工日、人工成本。
- 人员维度：员工参与项目、分项目工日、人工成本。
- 统计前提：周表必须已经通过或被锁定/汇总，退回和草稿不进入成本统计。

## 6. 权限模型

```mermaid
flowchart TD
  ROLE["user_roles.role"] --> EMPLOYEE["employee"]
  ROLE --> MANAGER["manager"]
  ROLE --> ADMIN["admin"]

  EMPLOYEE --> SELF_TS["本人周表读写<br/>draft / rejected 可编辑"]
  MANAGER --> REVIEW["审批中心与 BI 读取"]
  MANAGER --> MANAGED_EMP["负责部门及下级部门员工维护"]
  ADMIN --> ALL_DATA["全局读写 / 员工组织项目维护"]

  WHITE["历史白名单超级用户"] --> ADMIN_VIEW["应用层 admin 视图"]
```

### 员工与组织页当前规则

- admin：看全部员工和全部部门，可以新增/删除员工、维护组织、修改角色。
- 部门负责人：只看自己负责部门及下级部门的员工和部门，可以编辑这些员工的基础信息、合同、薪酬，但不能改系统角色。
- 普通员工：不可进入员工与组织页。
- 历史白名单超级用户：应用层按 admin 视图处理。

## 7. 实时刷新与缓存

```mermaid
flowchart LR
  PG["PostgreSQL 表变更"] --> PUB["supabase_realtime publication"]
  PUB --> RT["Realtime"]
  RT --> FE["useRealtime"]
  FE --> QC["React Query invalidate"]
  QC --> MODULES["timesheet / approvals / reports / dashboard / employees"]
```

表到前端模块的典型映射：

- `timesheets`：刷新周表、审批、报表、看板。
- `timesheet_entries`：刷新周表、报表、看板。
- `workflow_tasks`：刷新审批和看板。
- `employees` / `employee_profiles_v2` / `organizations`：刷新员工组织、审批、看板。
